Getting Started with ZingChart

Overview

ZingChart enables you to create fast, modern, powerful charts for web applications. This quick start document will help you build your first ZingChart and point you in the right direction for further exploration.

Requirements

A modern Web browser with Adobe Flash player 9+ is needed to view charts and use online builder, otherwise no particular IDE or programming environment is required to develop with ZingChart.

Quick Start

  1. Extract package files into a Web served directory

    Tip While you can simply access files via a local drive path, given same origin access controls and other browser security restrictions it is preferred to develop using a Web server.

  2. Create a sample file named hellozing.html and include the following code in the <head> tag

                        <script type="text/javascript" src="zingchart-1.1.js"></script>
                    <script type="text/javascript">
                    window.onload = function(){
                    	zingchart.render({
                    		liburl 			: 'zingchart.swf',
                    		dataurl 		: 'graphfile.txt',
                    		width	 		: 460,
                    		height 			: 460,
                    		container 		: 'zingchart'
                    	});
                    }
                    </script>

    Note Your paths may be different depending on where you installed the scripts.

  3. In hellozing.html create a <div> holder for the chart where you want the graph to be displayed.

                        <div id="zingchart"></div>

  4. Create the graphfile referenced above called graphfile.txt which will contain JSON for your first chart.

                        {
                      "graphset" : [
                      {
                        "type" : "line",
                        "series" : [
                         {
                            "values" : [5, 10, 15, 5, 10, 5]
                         }] 
                      }]
                    }

  5. Load hellozing.html in your browser and you're on your way.

Tip There are numerous ways to include ZingChart configuration including:

  • Internal JSON variable
  • External JSON config packet
  • Script API using the ZcData() method


You also may use simple scripts to bind to HTML tables or integrate into server-side scripts.

Next Steps with ZingChart

There are many ways you can build great charts in ZingChart and with literally hundreds of options we have, you can design the coolest (or ugliest) charts on the web. Before building more complex charts, familiarize yourself with the online builder, docs and sample charts.

    Builder
    The ZingChart builder allows you to toggle hundreds of features and attributes and view both the visual result and JSON code in real time. With the JSON Edit Mode, you can make manual attribute and data changes in addition to using the menu-style builder. Click View in Web Page to see an output of the chart. The builder is a great place to start interactively explore the features of ZingChart or even build a full chart directly. You can always make finishing tweaks to the code after you've copied the JSON into your HTML/.txt file.

    Docs
    The docs on zingchart.com make building and editing JSON easy. We've sorted all the charts, topics, and features into menus so you can directly access the specific attributes you're looking for and drop them directly into your code.

    Sample Charts
    We've included a few sample charts in the "examples" folder of your download package to get you started. You may alter the data and JSON attributes however you see fit, and use any of these examples as a platform to build your charts, or just use them as a reference for how to code different styles and features. Both the HTML and .txt files for each of these charts have been included in your download package.

    • Submitting Bugs

    As with any awesome software product, you may occasionally be the lucky person who discovers a bug. If so, don't panic; submitting bugs with ZingChart is quick and easy. Of course if you're using the free trial you can panic a little bit, because only licensed developers get full support. (Sorry we gotta eat too) Here are a few ways to notify us of bugs:

    Submit Bug Report Tool
    Right-click and select Submit Bug Report on any chart (must be hosted online). Add in a quick comment and your email to be notified when the issue has been resolved.

    Submit a Request Online
    You may submit a bug or feature request using our standard support form on the ZingChart website.

    Email Us
    You may submit a bug or feature request by emailing us at support@zingchart.com.

    Tweet Us
    While we'd love you to show us love on twitter, you can also request help or share ideas by mentioning @ZingChart or including the #zingchart hashtag in your tweets. Regardless, follow us for cool updates, charts, and other nonsense.
